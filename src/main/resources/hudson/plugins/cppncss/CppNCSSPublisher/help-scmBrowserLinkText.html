If this field is not empty, the report will include links to your git code repository browser. Use the variable names $file, $hash and $line where the file path, git commit hash and line number should go respectively. For example, for Stash/Bitbucket server, you would write the link as

http://<your-stash-url.com>/<stash-project>/repos/<git-repo>/browse/$file?at=$hash#$line

Your complexity report must contain file paths which are relative to the module root for this to work correctly.

Note: In a multibranch pipeline job, the variable $hash will not work. Instead, use environment variables. For example:

scmBrowserLinkText: 'http://blabla.com/?file=$file;git-hash=' + ${env.GIT_COMMIT} + ';id=$id'

The above works for a declarative pipeline. For a scripted pipeline, use smVars = checkout scm, and replace ${env.GIT_COMMIT} above by ${scmVars.GIT_COMMIT}
